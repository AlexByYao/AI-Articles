## C++（1-50）

### 简单（1-18）

1. **Q:** `new` 和 `malloc` 的区别是什么？
   **A:** `new` 是 C++ 运算符：会调用构造函数、返回强类型指针、失败抛异常；`malloc` 是 C 函数：只分配原始内存、返回 `void*`、失败返回 `nullptr`。
   **解释：** C++ 对象需要完整的构造/析构生命周期管理，因此 `new/delete` 与对象模型天然匹配；`malloc/free` 只处理内存。

2. **Q:** 指针与引用的区别？
   **A:** 引用必须初始化、不可重新绑定，语义上是“别名”；指针可为空、可重新指向。
   **解释：** 引用更安全、更适合参数传递；指针适合需要“可空”或“可变指向”的场景。

3. **Q:** `const` 成员函数的意义？
   **A:** 保证函数不修改对象的逻辑状态（`this` 为 `const T*`）。
   **解释：** 允许在 `const` 对象上调用该函数，同时使接口语义更清晰。

4. **Q:** RAII 是什么？
   **A:** 资源获取即初始化，在构造时获取资源，析构时释放。
   **解释：** 依靠栈对象自动析构，天然防止泄漏，如 `std::lock_guard`、`std::unique_ptr`。

5. **Q:** `std::vector` 与 `std::array` 区别？
   **A:** `vector` 动态长度、堆分配；`array` 固定长度、栈存储。
   **解释：** `array` 更轻量适合小固定集合；`vector` 适用于可变规模。

6. **Q:** `auto` 类型推导规则？
   **A:** 类似模板推导：忽略顶层 `const`，引用性需显式 `auto&`。
   **解释：** `auto` 提升可读性，但要注意推导结果与预期一致。

7. **Q:** `nullptr` 与 `NULL` 区别？
   **A:** `nullptr` 是 `std::nullptr_t` 类型，类型安全；`NULL` 通常是 0 或 `((void*)0)`。
   **解释：** 使用 `nullptr` 可避免重载歧义。

8. **Q:** `std::string` 与 C 风格字符串差异？
   **A:** `std::string` 管理内存、支持长度/拼接；C 字符串以 `\0` 结尾，需要手动管理。
   **解释：** C++ 更推荐 `std::string`，安全且易用。

9. **Q:** 结构体和类的区别？
   **A:** 默认访问权限不同：`struct` 默认 `public`，`class` 默认 `private`。
   **解释：** 语义上 `struct` 多用于数据聚合，`class` 多用于封装。

10. **Q:** `explicit` 关键字作用？
    **A:** 禁止单参数构造函数的隐式转换。
    **解释：** 避免隐式类型转换引发的隐性 bug。

11. **Q:** `static` 成员变量的意义？
    **A:** 属于类本身，所有对象共享一份。
    **解释：** 常用于计数器、共享配置等场景。

12. **Q:** `delete` 和 `delete[]` 的区别？
    **A:** `delete[]` 会对数组每个元素调用析构；`delete` 只对单对象。
    **解释：** 错用会产生未定义行为（UB）。

13. **Q:** STL 迭代器是什么？
    **A:** 容器元素的统一访问接口，类似指针。
    **解释：** 支持算法与容器解耦。

14. **Q:** `std::map` 和 `std::unordered_map` 区别？
    **A:** `map` 有序、红黑树实现 O(log n)；`unordered_map` 哈希实现，平均 O(1)。
    **解释：** 需要有序遍历或范围查询时选 `map`。

15. **Q:** 何为“对象切片”？
    **A:** 以值传递/赋值基类对象时丢失派生类部分。
    **解释：** 需用指针/引用保持多态。

16. **Q:** `sizeof` 是运行期还是编译期？
    **A:** 编译期常量表达式。
    **解释：** 可用于数组大小等编译期场景。

17. **Q:** `std::cout` 与 `printf` 区别？
    **A:** `cout` 是类型安全的流，支持重载；`printf` 依赖格式字符串。
    **解释：** `cout` 更安全但有时更慢。

18. **Q:** 什么是“未定义行为（UB）”？
    **A:** 违反语言规则，结果不可预测。
    **解释：** 例如越界访问、悬空引用等。

### 稍难（19-36）

19. **Q:** `std::unique_ptr` 与 `std::shared_ptr` 区别？
    **A:** `unique_ptr` 独占所有权，不可拷贝；`shared_ptr` 共享引用计数。
    **解释：** 共享指针有开销且可能循环引用，需谨慎。

20. **Q:** `std::weak_ptr` 用途？
    **A:** 解决 `shared_ptr` 循环引用。
    **解释：** 弱引用不增加计数，访问时需 `lock()`。

21. **Q:** 右值引用与移动语义的作用？
    **A:** 允许转移资源以避免深拷贝。
    **解释：** 典型如 `std::vector` 扩容时的移动构造。

22. **Q:** `std::move` 做了什么？
    **A:** 把对象强制转换为右值引用。
    **解释：** 并不移动资源，移动发生在移动构造/赋值中。

23. **Q:** `std::forward` 的用途？
    **A:** 完美转发，保留左值/右值属性。
    **解释：** 常用于模板函数。

24. **Q:** 虚函数表（vtable）机制？
    **A:** 多态类包含 vptr，指向 vtable，运行时动态分发。
    **解释：** 带来一点空间和间接调用成本。

25. **Q:** 为什么基类需要虚析构？
    **A:** 通过基类指针删除派生对象时确保析构链完整。
    **解释：** 否则派生资源可能泄漏。

26. **Q:** `override` / `final` 的意义？
    **A:** `override` 保证正确覆盖；`final` 禁止继承或重写。
    **解释：** 提高代码安全性和意图表达。

27. **Q:** 迭代器失效的常见场景？
    **A:** `vector` 扩容、`erase` 删除、容器结构变化。
    **解释：** 使用后需重新获取迭代器。

28. **Q:** `constexpr` 与 `const` 的区别？
    **A:** `constexpr` 强调编译期常量；`const` 只保证只读。
    **解释：** `constexpr` 可用于模板参数、数组大小。

29. **Q:** `noexcept` 的作用？
    **A:** 声明不抛异常，优化移动操作。
    **解释：** 如果抛异常会调用 `std::terminate`。

30. **Q:** 异常安全三种保证？
    **A:** 基本保证、强保证、不抛保证。
    **解释：** 可通过 RAII、复制交换等实现强保证。

31. **Q:** `std::function` 的开销来源？
    **A:** 类型擦除与可能的堆分配。
    **解释：** 若性能敏感可用模板或 `auto`。

32. **Q:** `std::optional` 适用场景？
    **A:** 可选返回值或“可能为空”的值类型。
    **解释：** 比裸指针表达更清晰。

33. **Q:** `std::variant` 解决什么问题？
    **A:** 类型安全的联合体。
    **解释：** 用 `std::visit` 实现模式匹配。

34. **Q:** `std::atomic` 用途？
    **A:** 提供无锁原子操作。
    **解释：** 避免数据竞争，但仍需理解内存序。

35. **Q:** `volatile` 的真实意义？
    **A:** 防止编译器对该变量做某些优化。
    **解释：** 不等于线程同步。

36. **Q:** 什么是“零成本抽象”？
    **A:** 抽象不增加额外运行开销。
    **解释：** C++ 模板与内联支持这一理念。

### 较难（37-50）

37. **Q:** 模板特化与偏特化区别？
    **A:** 全特化指定全部参数；偏特化只指定部分。
    **解释：** 用于类型相关优化或定制行为。

38. **Q:** `decltype` 与 `auto` 区别？
    **A:** `decltype` 保留表达式的精确类型（含引用/const）；`auto` 类似模板推导。
    **解释：** 用于复杂返回类型推导。

39. **Q:** `enable_shared_from_this` 为什么需要？
    **A:** 让对象安全获取自身的 `shared_ptr`。
    **解释：** 避免从 `this` 构造 `shared_ptr` 导致双重计数。

40. **Q:** C++ 内存对齐是什么？
    **A:** 对象地址需满足对齐要求，提高访问效率。
    **解释：** `alignas` 可控制对齐。

41. **Q:** `std::vector` 何时用 `reserve()`？
    **A:** 预知元素数量时。
    **解释：** 减少扩容与搬移成本。

42. **Q:** `std::sort` 的实现策略？
    **A:** 通常使用 introsort（快排+堆排+插排）。
    **解释：** 平均 O(n log n)，最坏也可保证。

43. **Q:** “浅拷贝/深拷贝”影响？
    **A:** 浅拷贝会共享资源，可能双重释放；深拷贝复制资源。
    **解释：** 资源类需要明确拷贝语义。

44. **Q:** `new` 失败如何处理？
    **A:** 默认抛 `std::bad_alloc`，也可用 `std::nothrow` 返回空指针。
    **解释：** 异常处理更安全。

45. **Q:** C++ 中的对象生命周期？
    **A:** 构造开始到析构结束。
    **解释：** 使用生命周期外对象会 UB。

46. **Q:** `std::vector<bool>` 特殊性？
    **A:** 位压缩特化，返回代理对象而非 `bool&`。
    **解释：** 可能导致接口陷阱。

47. **Q:** 什么是类型擦除？
    **A:** 用统一接口隐藏具体类型。
    **解释：** 如 `std::function`、`std::any`。

48. **Q:** `std::any` 与 `void*` 区别？
    **A:** `any` 保存类型信息，`void*` 无类型安全。
    **解释：** `any_cast` 可检测类型不匹配。

49. **Q:** 线程安全与可重入的区别？
    **A:** 线程安全指多线程下正确；可重入要求函数被中断/嵌套调用仍正确。
    **解释：** 可重入更严格。

50. **Q:** C++ 资源管理常见原则？
    **A:** Rule of 3/5/0。
    **解释：** 管理资源的类应正确实现拷贝/移动/析构或禁止拷贝。

## Qt（51-100）

### 简单（51-68）

51. **Q:** Qt 信号与槽机制是什么？
    **A:** 对象间通信机制，信号发出后调用连接的槽。
    **解释：** 松耦合、事件驱动，降低模块依赖。

52. **Q:** `QObject` 的作用？
    **A:** 提供元对象系统、信号槽、对象树、属性等。
    **解释：** Qt 大多数类继承自 `QObject`。

53. **Q:** `Q_OBJECT` 宏的意义？
    **A:** 启用 MOC 生成元对象代码。
    **解释：** 没有它无法使用信号槽与属性系统。

54. **Q:** Qt 对象树的好处？
    **A:** 父对象析构自动删除子对象。
    **解释：** 简化内存管理，减少泄漏。

55. **Q:** `deleteLater()` 与 `delete` 区别？
    **A:** `deleteLater()` 在事件循环安全时删除。
    **解释：** 避免在槽函数中直接 `delete` 造成崩溃。

56. **Q:** `QTimer` 典型用途？
    **A:** 计时器触发周期任务。
    **解释：** 依赖事件循环。

57. **Q:** `update()` 与 `repaint()` 区别？
    **A:** `update()` 延迟合并重绘，`repaint()` 立即绘制。
    **解释：** 一般使用 `update()` 更高效。

58. **Q:** `QWidget` 与 `QMainWindow` 区别？
    **A:** `QMainWindow` 提供菜单、工具栏、状态栏布局。
    **解释：** 适合主窗口应用。

59. **Q:** Qt 常用布局管理器？
    **A:** `QHBoxLayout`、`QVBoxLayout`、`QGridLayout`、`QFormLayout`。
    **解释：** 自动管理控件大小与位置。

60. **Q:** `QString` 与 `std::string` 区别？
    **A:** `QString` 用 UTF-16，便于国际化；`std::string` 是字节序列。
    **解释：** 转换需注意编码。

61. **Q:** Qt 国际化基本流程？
    **A:** `tr()` 标记 → `lupdate` 生成 ts → 翻译 → `lrelease` 生成 qm。
    **解释：** 运行时用 `QTranslator` 加载。

62. **Q:** Qt 中样式表（QSS）作用？
    **A:** 类似 CSS 的方式设置控件样式。
    **解释：** 便于统一 UI 风格。

63. **Q:** `QPixmap` 与 `QImage` 区别？
    **A:** `QPixmap` 适合屏幕显示（GUI 相关）；`QImage` 适合像素处理。
    **解释：** 多线程处理图片通常用 `QImage`。

64. **Q:** `QSettings` 用于什么？
    **A:** 持久化应用配置（注册表/ini）。
    **解释：** 跨平台一致接口。

65. **Q:** `QDialog` 的 `exec()` 与 `show()` 区别？
    **A:** `exec()` 模态阻塞；`show()` 非模态。
    **解释：** 需要等待用户决策时用 `exec()`。

66. **Q:** Qt 的事件过滤器怎么用？
    **A:** 继承 `QObject` 重载 `eventFilter`，用 `installEventFilter` 安装。
    **解释：** 可拦截/修改事件流程。

67. **Q:** Qt UI 线程规则？
    **A:** 所有 UI 必须在主线程创建和访问。
    **解释：** 跨线程访问会导致不稳定。

68. **Q:** `QFile` 与 C++ 文件流区别？
    **A:** `QFile` 与 Qt 生态一致，支持资源系统。
    **解释：** Qt 项目中通常使用 `QFile`。

### 稍难（69-86）

69. **Q:** `Qt::AutoConnection` 的行为？
    **A:** 同线程为直接连接，不同线程为队列连接。
    **解释：** 默认安全配置。

70. **Q:** 直接连接与队列连接区别？
    **A:** 直接连接立即调用槽；队列连接异步投递到目标线程。
    **解释：** 跨线程通信应使用队列连接。

71. **Q:** `moveToThread()` 的意义？
    **A:** 将 `QObject` 绑定到目标线程，使其槽在该线程执行。
    **解释：** 对象必须没有父对象。

72. **Q:** Qt 多线程推荐实践？
    **A:** `QThread + QObject` 或 `QtConcurrent`。
    **解释：** 避免直接在 `QThread::run()` 中写复杂逻辑。

73. **Q:** `QMetaObject::invokeMethod` 用途？
    **A:** 跨线程调用槽或函数，可指定连接方式。
    **解释：** 常用于线程间安全调用。

74. **Q:** `QVariant` 的作用？
    **A:** 通用类型容器，存储多种 Qt 类型。
    **解释：** 常用于模型/属性系统。

75. **Q:** Qt MVC 常用类？
    **A:** `QAbstractItemModel` + `QTableView/QListView`。
    **解释：** 数据与视图解耦。

76. **Q:** `QAbstractListModel` 如何实现？
    **A:** 重写 `rowCount()` 和 `data()`，必要时实现 `roleNames()`。
    **解释：** 适配 QML 或复杂视图。

77. **Q:** Qt 中信号槽连接失败原因？
    **A:** 签名不匹配、缺少 `Q_OBJECT`、对象已销毁。
    **解释：** `QObject::connect` 返回值可检查。

78. **Q:** Qt 的资源系统（.qrc）优势？
    **A:** 资源打包进可执行文件，部署方便。
    **解释：** 路径以 `:/` 开头。

79. **Q:** `QPainter` 使用注意点？
    **A:** 在 `paintEvent()` 中使用，避免跨线程绘制。
    **解释：** 频繁绘制需减少临时对象创建。

80. **Q:** `QGraphicsView` 体系用途？
    **A:** 适合复杂 2D 场景管理。
    **解释：** 提供 View/Scene/Item 架构。

81. **Q:** `QNetworkAccessManager` 的特性？
    **A:** 异步网络请求，信号槽回调。
    **解释：** 避免阻塞 UI 线程。

82. **Q:** Qt 中高 DPI 适配方法？
    **A:** 设置 `Qt::AA_EnableHighDpiScaling` 等属性。
    **解释：** 多屏缩放要注意逻辑像素。

83. **Q:** Qt 样式与皮肤自定义方式？
    **A:** QSS 或自定义 `QStyle`。
    **解释：** QSS 简单，`QStyle` 更灵活。

84. **Q:** `QElapsedTimer` 用途？
    **A:** 高精度计时。
    **解释：** 适合性能分析。

85. **Q:** Qt 中拖拽实现步骤？
    **A:** 开启 `setAcceptDrops(true)`，实现 `dragEnterEvent`/`dropEvent`。
    **解释：** 可自定义 MIME 类型。

86. **Q:** Qt 中快捷键管理？
    **A:** `QShortcut` 或 `QAction`。
    **解释：** 统一管理更易维护。

### 较难（87-100）

87. **Q:** QML 与 Widgets 的核心差异？
    **A:** QML 声明式、适合动画与复杂 UI；Widgets 命令式、传统桌面 UI。
    **解释：** 可混合，但要考虑架构复杂度。

88. **Q:** QML 与 C++ 交互方式？
    **A:** `qmlRegisterType`、`setContextProperty`、`Q_PROPERTY`。
    **解释：** 合理暴露 API 便于 UI 层调用。

89. **Q:** `QProperty`（Qt6）优势？
    **A:** 支持属性绑定与变更通知。
    **解释：** 更适配声明式 UI。

90. **Q:** `QThread` 继承与“对象移动”方式差别？
    **A:** 继承 `QThread` 重写 `run()` 无事件循环；移动对象方式更符合 Qt 信号槽模型。
    **解释：** 推荐“对象移动”方式。

91. **Q:** Qt 程序卡顿常见原因与优化？
    **A:** UI 线程阻塞、频繁重绘、同步 I/O。
    **解释：** 用线程/异步、合并更新、减少绘制开销。

92. **Q:** `Qt::QueuedConnection` 为什么线程安全？
    **A:** 槽调用被投递到目标线程事件队列。
    **解释：** 避免跨线程直接调用。

93. **Q:** Qt 中如何安全更新 UI（跨线程）？
    **A:** 用信号槽或 `invokeMethod` 投递到主线程。
    **解释：** 不要直接在子线程操作控件。

94. **Q:** Qt 中模型数据更新的最佳实践？
    **A:** 使用 `beginInsertRows/endInsertRows` 等通知视图。
    **解释：** 保障视图状态一致。

95. **Q:** Qt 中表单校验方式？
    **A:** `QValidator`（如 `QIntValidator`）。
    **解释：** 输入阶段限制，减少后端校验压力。

96. **Q:** Qt 中事件类型常见有哪些？
    **A:** 鼠标、键盘、定时器、绘制、窗口大小变化等。
    **解释：** 可重写具体事件函数或 `event()` 统一处理。

97. **Q:** `QApplication` 与 `QCoreApplication` 区别？
    **A:** 前者支持 GUI，后者用于非 GUI 程序。
    **解释：** GUI 程序必须用 `QApplication`。

98. **Q:** Qt 中自定义控件的关键点？
    **A:** 重写 `paintEvent()`，处理事件，结合布局与属性。
    **解释：** 可结合 QSS/Style 提升可维护性。

99. **Q:** Qt 如何避免内存泄漏？
    **A:** 使用父子对象机制、智能指针、`deleteLater()`。
    **解释：** QObject 树能自动管理资源。

100. **Q:** Qt 中跨线程通信的最佳方式？
     **A:** 信号槽（队列连接）+ 线程安全数据结构。
