# C++ / Qt 面试题汇总（2026-03-03）

> 说明：共 100 题，前 50 题为 C++ 核心，后 50 题为 Qt 方向；每题包含答案与简要解释。

## C++（1-50）

1. **Q:** `new` 和 `malloc` 的区别是什么？
   **A:** `new` 会调用构造函数并返回强类型指针，`malloc` 只分配原始内存并返回 `void*`。`new` 失败抛异常，`malloc` 返回 `nullptr`。`new/delete` 可重载，`malloc/free` 不能。解释：C++ 对象的生命周期管理需要构造/析构，`new/delete` 与此契合。

2. **Q:** 指针与引用的区别？
   **A:** 引用必须初始化且不可再绑定，语义更像别名；指针可为空、可重新指向。解释：引用更安全，但需要确保被引用对象生命周期长于引用本身。

3. **Q:** RAII 是什么？
   **A:** 资源获取即初始化，资源在对象构造时获取，析构时释放。解释：利用栈对象自动析构来管理资源，避免泄漏（如 `std::lock_guard`、`std::unique_ptr`）。

4. **Q:** `const` 在成员函数后面的意义？
   **A:** 表示该成员函数不会修改对象的逻辑状态（`this` 指针为 `const T*`）。解释：可用于区分重载，提高接口可读性。

5. **Q:** `constexpr` 和 `const` 区别？
   **A:** `constexpr` 目标是编译期常量，可用于数组大小、模板参数；`const` 只是运行期只读。解释：`constexpr` 函数在可推导时编译期执行。

6. **Q:** 右值引用与移动语义的作用？
   **A:** 右值引用（`T&&`）允许“偷资源”，减少拷贝成本。解释：移动构造/赋值可转移堆内存所有权，提高性能。

7. **Q:** 什么时候需要自定义拷贝构造/赋值？
   **A:** 当类管理资源（裸指针、句柄）且需要深拷贝或禁用拷贝时。解释：遵循“Rule of 3/5/0”。

8. **Q:** `std::unique_ptr` 和 `std::shared_ptr` 区别？
   **A:** `unique_ptr` 独占所有权不可拷贝；`shared_ptr` 共享引用计数。解释：共享会引入计数开销与循环引用风险。

9. **Q:** `std::weak_ptr` 解决什么问题？
   **A:** 解决 `shared_ptr` 循环引用导致的内存泄漏。解释：弱引用不增加计数，可在访问前 `lock()`。

10. **Q:** 什么是虚函数表（vtable）？
    **A:** 多态类含虚函数表指针，运行时通过 vtable 进行动态分发。解释：增加一点空间开销换取多态能力。

11. **Q:** 虚析构函数为何重要？
    **A:** 通过基类指针删除派生对象时，确保派生类析构被调用。解释：否则可能资源泄漏。

12. **Q:** `override` 和 `final` 的用途？
    **A:** `override` 强制检查是否真正覆盖基类虚函数；`final` 禁止进一步继承或覆盖。解释：提高安全性和可读性。

13. **Q:** STL 中 `vector` 的扩容机制？
    **A:** 通常按倍增策略扩容，导致迭代器失效。解释：扩容时重新分配并搬移元素。

14. **Q:** `std::map` 与 `std::unordered_map` 区别？
    **A:** `map` 基于红黑树，有序 O(log n)；`unordered_map` 基于哈希，平均 O(1)。解释：有序适合范围查询。

15. **Q:** 迭代器失效的常见场景？
    **A:** `vector` 扩容、`erase` 删除元素等。解释：容器修改结构时可能使迭代器指向无效地址。

16. **Q:** `std::move` 做了什么？
    **A:** 只是把对象强制转换为右值引用。解释：真正的资源移动发生在移动构造/赋值中。

17. **Q:** `std::forward` 的作用？
    **A:** 完美转发，保留参数的左/右值属性。解释：模板中常配合转发引用使用。

18. **Q:** 线程安全与“可重入”区别？
    **A:** 线程安全指多线程下行为正确；可重入指同一函数被中断/嵌套调用仍正确。解释：可重入通常更严格。

19. **Q:** `std::mutex` 与 `std::recursive_mutex` 区别？
    **A:** `recursive_mutex` 允许同线程多次加锁。解释：一般不推荐滥用，可能隐藏设计问题。

20. **Q:** `std::atomic` 解决什么问题？
    **A:** 提供无锁原子操作，避免数据竞争。解释：需理解内存序（`memory_order`）。

21. **Q:** `volatile` 在 C++ 中的意义？
    **A:** 告诉编译器该变量可能被外部因素修改，禁止特定优化。解释：**不**等同线程同步。

22. **Q:** `static` 在类成员中的意义？
    **A:** 类共享成员，不属于对象实例。解释：静态成员函数无法访问非静态成员。

23. **Q:** `friend` 的作用和风险？
    **A:** 允许特定类/函数访问私有成员。解释：打破封装，需谨慎使用。

24. **Q:** 什么是“对象切片”？
    **A:** 通过值传递或拷贝基类对象时丢失派生部分。解释：应使用引用或指针。

25. **Q:** `explicit` 关键字的作用？
    **A:** 防止单参数构造函数的隐式转换。解释：避免意外类型转换导致的 bug。

26. **Q:** `noexcept` 的意义？
    **A:** 声明函数不抛异常，可优化移动操作与异常安全。解释：抛出异常会触发 `std::terminate`。

27. **Q:** 异常安全的三种保证？
    **A:** 基本保证（无资源泄漏）、强保证（操作失败不改变状态）、不抛保证。解释：通过 RAII 和复制交换实现。

28. **Q:** `std::string` 的小字符串优化（SSO）是什么？
    **A:** 短字符串存储在栈内存中，不分配堆。解释：减少小字符串的分配开销。

29. **Q:** 模板特化与偏特化区别？
    **A:** 全特化指定所有模板参数；偏特化只限定一部分参数。解释：用于不同类型的优化实现。

30. **Q:** `decltype` 的作用？
    **A:** 推导表达式的类型。解释：常配合 `auto`、模板返回类型推导使用。

31. **Q:** `auto` 的类型推导规则？
    **A:** 类似模板推导，忽略顶层 `const`，引用性需显式 `auto&`。解释：可读性与编译期类型推断。

32. **Q:** `std::bind` 与 `lambda` 的区别？
    **A:** `lambda` 更直观、类型安全；`bind` 语法较晦涩。解释：现代 C++ 推荐 `lambda`。

33. **Q:** `std::function` 的开销？
    **A:** 类型擦除带来一定运行时开销。解释：可用模板或 `auto` 减少开销。

34. **Q:** `std::variant` 的用途？
    **A:** 安全的联合体类型，存储多种可能类型。解释：用 `std::visit` 访问。

35. **Q:** `std::optional` 适用场景？
    **A:** 可选返回值或可为空的值类型。解释：比裸指针表达意图更清晰。

36. **Q:** `std::any` 和 `void*` 区别？
    **A:** `any` 带类型信息和安全检查；`void*` 无类型安全。解释：`any_cast` 失败会抛异常或返回空。

37. **Q:** 解释“零成本抽象”。
    **A:** 抽象不应带来运行时开销（或与手写等价）。解释：模板和内联支持此理念。

38. **Q:** 什么是“未定义行为（UB）”？
    **A:** 代码违反语言规则导致结果不可预测。解释：例如越界访问、使用悬空指针。

39. **Q:** `std::vector<bool>` 的特殊性？
    **A:** 使用位压缩特化，返回代理对象而非 `bool&`。解释：可能导致接口陷阱。

40. **Q:** 解释“类型擦除”。
    **A:** 用统一接口隐藏具体类型。解释：`std::function`、`std::any` 都是例子。

41. **Q:** `new` 失败的处理方式？
    **A:** 默认抛出 `std::bad_alloc`；可用 `std::nothrow` 返回空指针。解释：推荐异常处理。

42. **Q:** `delete[]` 和 `delete` 的区别？
    **A:** `delete[]` 释放数组并调用每个元素析构。解释：错用会造成 UB。

43. **Q:** 什么是“内存对齐”？
    **A:** 对象地址按特定字节对齐，提高访问效率。解释：`alignas` 可指定对齐。

44. **Q:** `std::sort` 的时间复杂度？
    **A:** 平均 O(n log n)，通常使用 introsort。解释：大多数场景性能优良。

45. **Q:** 解释“浅拷贝”和“深拷贝”。
    **A:** 浅拷贝只复制指针；深拷贝复制指向的数据。解释：资源类应实现深拷贝或禁用拷贝。

46. **Q:** `const` 成员变量如何初始化？
    **A:** 必须在构造函数初始化列表中。解释：构造体内不能再赋值。

47. **Q:** 解释“左值/右值”。
    **A:** 左值有持久身份与地址，右值通常是临时对象。解释：决定绑定到引用的规则。

48. **Q:** `std::enable_shared_from_this` 的用途？
    **A:** 让对象内部安全地获取自身的 `shared_ptr`。解释：避免从 `this` 直接构造导致双重计数。

49. **Q:** C++ 中的“对象生命周期”是什么？
    **A:** 从构造开始到析构结束。解释：访问已析构对象会导致 UB。

50. **Q:** `std::vector` 何时适合用 `reserve()`？
    **A:** 已知元素数量时，提前分配避免多次扩容。解释：降低拷贝/移动成本。

## Qt（51-100）

51. **Q:** Qt 的信号与槽机制是什么？
    **A:** 对象间通信机制，信号发出后自动调用已连接的槽函数。解释：实现松耦合、事件驱动。

52. **Q:** Qt 中 `QObject` 的作用？
    **A:** 提供元对象系统、信号槽、对象树、属性系统等。解释：大多数 Qt 类继承自 `QObject`。

53. **Q:** `Q_OBJECT` 宏的作用？
    **A:** 启用元对象系统（MOC 生成代码）。解释：没有它无法使用信号槽和属性系统。

54. **Q:** Qt 的对象树有何意义？
    **A:** 父对象析构时自动析构子对象。解释：简化内存管理，避免泄漏。

55. **Q:** `deleteLater()` 与 `delete` 的区别？
    **A:** `deleteLater()` 在事件循环安全时删除对象，避免正在使用时销毁。解释：常用于 QObject 在信号槽中删除。

56. **Q:** 直接连接与队列连接的区别？
    **A:** 直接连接在发射线程立即调用槽；队列连接跨线程异步调用。解释：线程间通信更安全。

57. **Q:** `Qt::AutoConnection` 的行为？
    **A:** 同线程为直接连接，不同线程为队列连接。解释：默认安全配置。

58. **Q:** `QTimer` 的典型使用场景？
    **A:** 计时触发事件或周期任务。解释：依赖事件循环。

59. **Q:** Qt 事件循环的核心类？
    **A:** `QEventLoop` / `QCoreApplication` / `QApplication`。解释：负责分发事件。

60. **Q:** `QWidget` 与 `QMainWindow` 区别？
    **A:** `QWidget` 是基础控件；`QMainWindow` 提供菜单栏、工具栏、状态栏布局。解释：适合主窗体应用。

61. **Q:** Qt 的布局管理器有哪些？
    **A:** `QHBoxLayout`、`QVBoxLayout`、`QGridLayout`、`QFormLayout`。解释：自动管理控件大小与位置。

62. **Q:** `QPainter` 的使用注意点？
    **A:** 必须在 `paintEvent()` 中使用，不能跨线程直接绘制。解释：绘制要尽量避免频繁分配。

63. **Q:** `update()` 与 `repaint()` 区别？
    **A:** `update()` 触发延迟重绘，合并多次请求；`repaint()` 立即重绘。解释：一般用 `update()` 更高效。

64. **Q:** Qt 的 MVC 结构（模型/视图）常用类？
    **A:** 模型 `QAbstractItemModel`，视图 `QTableView`、`QListView` 等。解释：数据与显示解耦。

65. **Q:** `QString` 与 `std::string` 区别？
    **A:** `QString` 用 UTF-16 存储，便于国际化；`std::string` 字节序列。解释：转换需注意编码。

66. **Q:** Qt 中如何实现国际化？
    **A:** 使用 `tr()` 标记字符串，借助 `lupdate/lrelease` 生成翻译文件。解释：动态切换语言需 `QTranslator`。

67. **Q:** Qt 的事件过滤器怎么用？
    **A:** 继承 `QObject` 并重载 `eventFilter`，通过 `installEventFilter` 安装。解释：可拦截与处理特定事件。

68. **Q:** 解释 `signals:`、`slots:` 关键字。
    **A:** 用于声明信号与槽，MOC 会生成相应元信息。解释：信号无实现，槽是普通成员函数。

69. **Q:** Qt 多线程推荐方式？
    **A:** `QThread` + `QObject` 移动到线程，或 `QtConcurrent`。解释：避免在子线程直接操作 UI。

70. **Q:** `moveToThread()` 的意义？
    **A:** 将对象绑定到指定线程，槽函数在该线程执行。解释：需确保对象无父对象。

71. **Q:** Qt 中 UI 线程规则？
    **A:** 所有 GUI 控件必须在主线程创建和访问。解释：跨线程访问会导致不可预期行为。

72. **Q:** `QMetaObject::invokeMethod` 用途？
    **A:** 跨线程调用槽或函数，支持队列连接。解释：常用于线程间通信。

73. **Q:** `QVariant` 的作用？
    **A:** 通用类型容器，可保存多种 Qt 类型。解释：常用于模型数据和属性系统。

74. **Q:** `QProperty`（Qt6）有何优势？
    **A:** 支持绑定与自动通知。解释：更接近声明式 UI 需求。

75. **Q:** Qt 的资源系统（.qrc）是什么？
    **A:** 将图片、文件等编译进可执行文件。解释：便于部署。

76. **Q:** `QPixmap` 与 `QImage` 区别？
    **A:** `QPixmap` 适合屏幕显示，依赖 GUI；`QImage` 适合像素处理与 IO。解释：线程安全性不同。

77. **Q:** `QFile` 与标准 C++ 文件流区别？
    **A:** `QFile` 与 Qt 生态一致，支持资源系统；C++ 流更底层。解释：Qt 代码统一使用 `QFile` 更便捷。

78. **Q:** `QSettings` 用于什么？
    **A:** 保存应用配置（注册表/ini）。解释：跨平台一致接口。

79. **Q:** `QDialog` 的 `exec()` 与 `show()` 区别？
    **A:** `exec()` 模态阻塞，`show()` 非模态。解释：需要交互结束再继续时用 `exec()`。

80. **Q:** `QDockWidget` 的作用？
    **A:** 可停靠/浮动的子窗口。解释：常用于 IDE 类应用。

81. **Q:** Qt 中样式表（QSS）作用？
    **A:** 类似 CSS 的方式设置控件样式。解释：可快速统一 UI 风格。

82. **Q:** `QAbstractListModel` 如何实现？
    **A:** 重写 `rowCount()` 和 `data()`，可添加 `roleNames()`。解释：适配 QML/视图。

83. **Q:** Qt 中信号槽连接失败常见原因？
    **A:** 签名不匹配、缺少 `Q_OBJECT`、对象已销毁。解释：使用 `QObject::connect` 返回值检查。

84. **Q:** Qt 的 `Qt::QueuedConnection` 线程安全原因？
    **A:** 事件被投递到目标线程事件队列。解释：避免直接跨线程调用。

85. **Q:** 如何避免 Qt 中内存泄漏？
    **A:** 使用父子对象机制或智能指针，及时 `deleteLater()`。解释：QObject 树可自动管理。

86. **Q:** `QGraphicsView` 体系的用途？
    **A:** 适合复杂 2D 图形场景与交互。解释：包含 View/Scene/Item 架构。

87. **Q:** `QElapsedTimer` 与 `QTime` 区别？
    **A:** `QElapsedTimer` 用于高精度耗时测量。解释：`QTime` 更偏向时间表示。

88. **Q:** Qt 中如何处理高 DPI？
    **A:** 使用 `Qt::AA_EnableHighDpiScaling` 等属性。解释：支持多屏不同缩放。

89. **Q:** Qt 的 `QML` 与 `Widgets` 区别？
    **A:** QML 声明式、适合复杂 UI 动画；Widgets 传统命令式。解释：不同场景选择不同技术栈。

90. **Q:** QML 与 C++ 交互方式？
    **A:** `QQmlContext::setContextProperty`、注册类型 `qmlRegisterType`。解释：暴露属性与槽供 QML 使用。

91. **Q:** `QThread` 直接子类化与“对象移动”方式差别？
    **A:** 子类化重写 `run()` 不方便信号槽；移动对象到线程更符合 Qt 推荐。解释：事件循环在 `QThread` 中。

92. **Q:** Qt 中 `QNetworkAccessManager` 的特点？
    **A:** 异步网络请求、信号槽回调。解释：适合 UI 程序避免阻塞。

93. **Q:** Qt 中如何做表单校验？
    **A:** `QValidator`（如 `QIntValidator`、`QRegExpValidator`）。解释：在输入阶段限制数据。

94. **Q:** Qt 中如何处理快捷键？
    **A:** 使用 `QShortcut` 或 `QAction` 绑定。解释：集中管理更易维护。

95. **Q:** Qt 的事件类型常见有哪些？
    **A:** 鼠标、键盘、定时器、绘制、窗口大小等。解释：继承 `QWidget` 重写 `event()` 或具体事件函数。

96. **Q:** `QApplication` 和 `QCoreApplication` 区别？
    **A:** `QApplication` 用于 GUI 程序；`QCoreApplication` 用于非 GUI。解释：后者不包含窗口系统。

97. **Q:** Qt 中如何实现拖拽？
    **A:** 处理 `dragEnterEvent`、`dropEvent`，设置 `setAcceptDrops(true)`。解释：可自定义 MIME 类型。

98. **Q:** `QStyle` 的作用？
    **A:** 控制控件渲染风格。解释：可实现自定义风格。

99. **Q:** Qt 程序卡顿常见原因与优化思路？
    **A:** UI 线程阻塞、频繁重绘、I/O 同步。解释：使用线程/异步、合并更新、减少绘制开销。

100. **Q:** Qt 中如何安全地在后台线程更新 UI？
     **A:** 使用信号槽或 `invokeMethod` 把更新投递到主线程。解释：避免直接跨线程操作控件。
